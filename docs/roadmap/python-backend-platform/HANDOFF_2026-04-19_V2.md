# Handoff 2026-04-19 V2

## Qué quedó hecho

- el roadmap principal quedó reconciliado contra el estado real del repo
- se cerró en código el mínimo operativo v1 para `analyze_cfdi`
- el backend ya publica fallback contractual con `current-ts`
- el request `xml` quedó acotado a `1_000_000` caracteres
- el timeout de providers quedó fijado en `15s`
- los errores fatales públicos quedaron neutrales y sin detalle interno
- se agregó `npm run test:api` como baseline de backend
- el backend ahora emite logs estructurados y métricas locales mínimas para `analyze_cfdi`
- request inválido y runtime fatal también quedan instrumentados con `requestId`

## Estado real al cierre

- el backend ya es owner de fallback governance
- el frontend ya no decide fallback sobre respuestas HTTP contractuales
- `fallbackReason` quedó cerrado a `provider_runtime_failure`
- el request inválido devuelve forma contractual segura
- `cfdi.findings` sigue siendo la fuente oficial, pero continúa sin findings ricos desde Python
- la observabilidad mínima ya no es solo semántica: existe localmente en backend y tests
- la política mínima de datos sensibles ya evita logging de XML y campos fiscales crudos en esta capa

## Qué sigue de verdad

El siguiente frente correcto es:

1. estabilizar v1 como capability operable
2. enriquecer `cfdi.findings`
3. reemplazar placeholders de `verdict` y `supportText`
4. evaluar retiro del fallback local visible
